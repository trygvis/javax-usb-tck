Running the Tests


Test your javax.usb implementation

To verify your javax.usb implementation against the java.usb TCK, you must run the following tests as specified and verify your results match the expected results listed with each test.

Notes: The tests in the test suite must be executed from the root directory of the extracted archive (e.g. /javaxusbtck-1_0/).

The tests are grouped into test programs based on hardware configuration and EEPROM image required to be programmed into the Cypress EZ-USB FX development board.

The EEPROM images required for each test are located in /javaxusbtck-1_0/images.

Isochronous and Default Control Pipe Image Tests

Bulk and Interrupt Image Tests

Note: As noted above, to pass the TCK all tests must run and produce the expected results as specified in this documentation. However, if you are doing preliminary testing with the TCK and run into errors, before running the next test make sure each USB device is reset to its initial state by unplugging and replugging the device into the USB system. Alternatively for the Cypress development board, pressing the button labeled "Reset" on the Cypress development board will cause the board to be reset to its initial state. The Cypress board has completed resetting when the digital counter starts incrementing for the isodpc.b6 and bulkint.b6 images. If the topology.b6 image is programmed into the board the counter is not used, so wait a few seconds after reset before running the tests.
