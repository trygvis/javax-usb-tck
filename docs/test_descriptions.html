JAVAX.USB TCK Tests
JAVAX.USB TCK Tests
Test Performs Tests
SignatureTest
  • This is the API for the entire javax.usb.* package.
  • Verifies existence of all expected public classes and interfaces.
  • Also verifies the existence, correct modifiers, arguments and throws clauses of public constructors and methods for each of the public classes and interfaces.
  • Verifies the existence and correct modifiers of public fields.
TopologyTest
  • Verifies that the API can traverse a hierarchical device topology.
  • Verifies that it properly reveals the correct pre-configured topology as defined by the test hardware setup.
DefaultControlPipeTest
  • Verifies that ControlTransfers work successfully on the DefaultControlPipe.
  • Verifies that proper events are generated and proper exceptions are thrown during the ControlTransfer operations.
Control I/O Test
  • Verifies pipes can be opened on a control endpoint other than endpoint 0.
  • Programmable device used in this test suite does not support control pipes other than the default control pipe.
  • Target device for this test will be any device with at least one non default control pipe.
  • Minimal communication verification will be performed with the non default control pipe endpoint since function will vary with device selected.
Bulk I/O Test
  • Verifies IN and OUT pipes can be opened and closed.
  • Verifies that Bulk operations work successfully, proper events are generated and proper executions are thrown in operation.
Interrupt I/O Test
  • Verifies IN and OUT pipes can be opened and closed.
  • Verifies that Interrupt operations work successfully, proper events are generated and proper executions are thrown in operation.
Isochronous I/O Test
  • Verifies IN and OUT pipes can be opened and closed.
  • Verifies that Isochronous operations work successfully, proper events are generated and proper executions are thrown in operation.
RequestTest
  • Verifies methods of the Standard Request class by creating a Request object.
  • Verifies methods of Standard Request send messages to the DefaultControlPipe.
IRPTest
  • Verifies the creation of IRP's and Control IRP's from the UsbDevice, UsbPipe, UsbDefaultIrp, UsbDefaultControlIrp classes.
ConstantsTest
  • Verifies constants defined in the implementation against those defined in the USB specification.
HotPlugTest
  • Verifies that proper UsbSevices and UsbDevice events are generated when devices are attached and detached at runtime.
Interface Policy Test
  • Verifies that the non-default UsbInterfacePolicy is being used correctly.

The full descriptions of each test are defined in the document Javax.usb TCK Test Suite
